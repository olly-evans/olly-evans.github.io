## Journey Programming a Chess Engine

I've learnt that the first step for implementing many of the algorithms and techniques I've heard about with regard to chess programming such as Minimax and Alpha-Beta pruning is in actual fact coding chess the game. Which has been much harder than I thought it would be initially but has  also been very rewarding. Thus, this post is split into two: programming the game and complying with the rules set by the FIDE and then eventually the actual engine.

If you're asking why I didn't use a prebuilt chess GUI its because of my limited Object-Oriented programming experience. And Chess is for sure a great way to practise this with its board and piece hierarchies. Also did it though because Dear Imgui in Visual Studio scares me to the core.
