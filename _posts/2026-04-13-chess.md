## Journey Programming a Chess Engine

I've learnt that the first step for implementing many of the algorithms and techniques I've heard about with regard to chess programming such as Minimax and Alpha-Beta pruning is in actual fact: coding chess the game believe it or not. Which has been much harder than I thought it would be initially, but has  also been very rewarding. Thus, this post is split into two: programming the game and complying with the rules set by the FIDE, and then eventually the actual engine.

If you're asking why I didn't use a prebuilt chess GUI its because of my limited Object-Oriented programming experience. And Chess is for sure a great way to practise this with its board and piece hierarchies. Also did it though because Dear Imgui in Visual Studio scares me to my core.

## Step 1: Programming the Game.

### Tools

To develop my C and OOP experience I chose to write this in C++. For the GUI and window handling I chose SFML because of its ease of use. 

One command and you can create windows and draw to them, its a welcome change from other more complex C++ GUIs I've tried.

```bash
sudo apt-get install libsfml-dev
```
